Lab Exercises

Lab 1: UI Engineering

Explorations in Jetpack Compose layout architectures and declarative view positioning configurations.

Through this lab, I learn layouts of Jetpack Compose, such as Columns, Rows, and Boxes, to build clean native user interfaces. This knowledge allowed me to structure the layout architecture for my application's Home Screen and custom components effectively. It taught me how to handle UI positioning reactively without relying on traditional XML layouts.

Lab 2: State & Interactions

Capturing explicit view events and handling mutation bindings cleanly within the pipeline lifecycle.

This lab enhanced my understanding of state management and event handling in Android using state primitives like remember and mutableStateOf. I successfully applied these interaction concepts to capture user inputs dynamically and toggle UI states in real time. It was crucial for enabling features like symptom selection, where the interface must immediately respond to user taps.

Lab 3: Material Design 3

Utilizing modern Design Tokens, Scaffolding architectures, and uniform context structuring frameworks.

Implementing Material Design 3 components and theming in this lab helped me understand how to create a visually appealing and accessible user experience. I learned how to utilize advanced design components like Scaffold, TopAppBar, and custom dynamic Card styling for my project. This directly improved the look and feel of my dashboard and symptom cards, ensuring they comply with modern Android design standards.

Lab 4: Backstack Navigation

Abstracting explicit component intent routing utilizing modern declarative structural frameworks.

This lab provided me with the skills to implement the Jetpack Compose Navigation component to manage screen routing and deep linkages. It helped me design a seamless 7-screen architecture for my project, allowing smooth user transitions from the dashboard to the history and locator screens. I also learned how to maintain a clean back-stack and prevent navigation lag between active destinations.

Lab 5: Room Architecture

Configuring DAOs, continuous reactive flows, and asynchronous system data transactions.

Through this lab, I learned how to set up local database persistence using the Room Library, including configuring Entities, DAOs, and Repositories. This allowed me to implement a local-first architecture for my application, enabling users to save and view their cycle history permanently without an internet connection. It also deepened my understanding of how to observe database tables reactively using Kotlin StateFlow to automatically update the UI layer.

Course Projects

Project 1: PeriodTracker Core

Problem Statement: Many women lack simple, accessible digital tools to track and understand their menstrual health patterns efficiently.

Solution: Developed a core Period Tracker application that enables users to effortlessly log critical health data such as dates, symptoms, and moods. By utilizing a shared ViewModel architecture, the app ensures seamless data communication across screens to provide comprehensive and immediate health insights.

Lessons Learned: Project 1 taught me the basics of building an Android app using Jetpack Compose and Kotlin. Project 2: PeriodTracker Advanced Ecosystem

Problem Statement: Women often struggle to maintain consistent menstrual health records due to fragmented digital resources. Furthermore, when irregularities or unexpected medical concerns arise, a critical gap exists in accessing immediate physical care because users lack real-time information to locate nearby healthcare facilities.

Solution: Expanded the app into an advanced ecosystem featuring a centralized tracking system for cycles, moods, and medical history. Integrated an intelligent Clinic Locator powered by GPS hardware and Retrofit web APIs to dynamically find nearby medical facilities. The system implements an offline-first architecture using Room Database for secure local storage, combined with Firebase Firestore for reliable real-time cloud backups.

Lessons Learned: In Project 2, I learned how to build a much bigger app by connecting a local Room Database with online Firebase cloud storage. I also gained great experience using the phone's GPS sensor and fetching live clinic data from the internet using Retrofit. This project taught me how to solve real-world health problems like SDG 3 using advanced coding tools.
